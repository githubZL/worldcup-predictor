# 世界杯预测控制台产品需求文档

## 1. 产品概述

### 1.1 产品名称

世界杯预测控制台

### 1.2 产品定位

面向世界杯观赛、赛前分析、赛果复盘和模型迭代的交互式预测工具。产品不是简单输出“谁会赢”，而是将赛程、比分预测、胜平负概率、半场/全场、半全场、受让球胜平负、竞彩数据校准、专家因子权重、场景修正和赛后复盘集中在一个可操作的仪表盘中。

### 1.3 核心目标

1. 帮用户快速查看当日重点赛事预测结论。
2. 解释预测依据，而不是只给最终比分。
3. 支持按赛程点击切换单场详情。
4. 支持结合当日竞彩数据，对模型概率进行校准和风险提示。
5. 支持赛后更新真实比分，复盘模型命中率和偏差。
6. 预留真实数据源接入能力，后续可扩展为自动更新系统。

### 1.4 当前阶段

最小可用产品原型阶段。

当前先完成静态/模拟数据版本，验证页面结构、信息架构、交互模式和预测展示逻辑。真实数据来源、接口稳定性、模型计算服务、竞彩数据授权方式将在后续讨论后补充。

## 2. 用户与使用场景

### 2.1 目标用户

1. 世界杯深度观众：希望赛前快速了解强弱、赔率变化、冷门风险。
2. 足球分析爱好者：关注战术、伤病、年龄、天气、海拔、比赛时段等因子。
3. 模型复盘用户：希望看到预测是否准确，以及模型长期表现。
4. 内容创作者：需要将复杂预测逻辑整理成可视化结论，用于视频、文章或社群分享。

### 2.2 典型场景

赛前查看：
用户打开页面，直接看到焦点战预测、预测比分、胜平负概率、竞彩校准、场景修正和当日赛程。

单场分析：
用户点击赛程中的某一场比赛，页面上方焦点战模块同步切换为该场比赛，并展示对应预测比分、半场比分、胜平负倾向、受让球倾向和关键影响因子。

竞彩校准：
用户查看胜平负、受让球胜平负、半全场三个市场的赔率、隐含概率、模型概率和差值，判断市场热度与模型分歧。

复盘：
比赛结束后，页面展示真实比分、预测比分、命中情况、模型误差和近 30 天复盘表现。

## 5. 功能需求

## 5.1 焦点战预测

### 功能说明

展示当前选中比赛的核心预测结论，是页面第一视觉重点。

### 展示字段

1. 比赛时间
2. 比赛阶段
3. 所属小组
4. 比赛场馆
5. 天气
6. 主队名称、国旗、世界排名
7. 客队名称、国旗、世界排名
8. 预测全场比分
9. 预测半场比分
10. 爆冷风险等级
11. 主胜、平局、客胜概率
12. 最可能比分 Top 3
13. xG 预测进球

### 交互模式

1. 默认展示当天最重要或最早的一场比赛。
2. 用户点击“今日赛程与预测”中的任意比赛，焦点战预测模块实时更新。
3. 被选中的赛程行高亮。
4. 概率条根据所选比赛动态变化。
5. 风险等级根据所选比赛动态变化。

### 状态规则

未赛：
展示预测比分、预测半场、概率、理由和风险。

已赛：
展示真实比分、预测比分、是否命中、误差和复盘标签。

延期/取消：
隐藏预测结论或显示“暂无预测”，并说明原因。

## 5.2 今日赛程与预测

### 功能说明

以表格形式展示当日或筛选范围内的全部比赛，并作为焦点战切换入口。

### 展示字段

1. 时间
2. 阶段
3. 组别
4. 对阵
5. 预测比分
6. 预测半场/全场
7. 胜平负倾向
8. 受让球倾向
9. 关键因子影响
10. 比赛状态

### 交互模式

1. 点击任意比赛行，更新上方焦点战预测。
2. 当前选中行显示高亮边框和背景。
3. 鼠标悬停行时显示浅色背景。
4. 表格横向空间不足时允许横向滚动。
5. 后续可支持点击行进入单场详情页。

### 筛选模式

顶部筛选项影响赛程列表：

1. 日期筛选：全部日期、今日、明日、指定日期。
2. 小组筛选：全部小组、A组、B组、C组等。
3. 球队筛选：全部球队、指定球队。
4. 阶段筛选：小组赛、淘汰赛、半决赛、决赛。

最小可用产品阶段可先展示控件，不实现复杂筛选；后续接入真实数据后实现。

## 5.3 竞彩校准

### 功能说明

结合当日竞彩数据，将市场赔率转换为隐含概率，并与模型概率进行对比，识别市场热度、模型分歧和风险。

### 子模块

1. 胜平负
2. 受让球胜平负
3. 半全场

### 胜平负字段

1. 选项：主胜、平、客胜
2. 开盘赔率
3. 当前赔率
4. 隐含概率
5. 模型概率
6. 差值
7. 趋势
8. 热度/风险

### 受让球胜平负字段

1. 让胜
2. 让平
3. 让负
4. 盘口信息，如“法国 +0.5”
5. 开盘赔率
6. 当前赔率
7. 隐含概率
8. 模型概率
9. 差值
10. 趋势

### 半全场字段

1. 胜胜
2. 胜平
3. 胜负
4. 平胜
5. 平平
6. 平负
7. 负胜
8. 负平
9. 负负
10. 隐含概率
11. 模型概率
12. 差值
13. 风险等级

### 交互模式

1. 顶部使用分段控件切换：胜平负、受让球胜平负、半全场。
2. 默认展示胜平负主表，受让球胜平负和半全场通过分段控件切换。
3. 差值为正且明显高于阈值时标红，表示市场热度高于模型判断或存在风险。
4. 差值为负时标绿，表示模型概率高于市场隐含概率或市场低估。
5. 赔率上升显示红色上箭头。
6. 赔率下降显示绿色下箭头。
7. 鼠标悬停某行时，后续可显示解释气泡。
8. 用户点击赛程切换比赛后，竞彩校准读取当前比赛对应的市场数据。
9. 若当前比赛缺少某类竞彩数据，对应分段控件置灰，并显示“暂无数据”。

### 计算逻辑

隐含概率：

```text
隐含概率 = 1 / 当前赔率
```

标准化隐含概率：

```text
标准化隐含概率 = 单项隐含概率 / 全部选项隐含概率之和
```

差值：

```text
差值 = 模型概率 - 标准化隐含概率
```

风险等级建议：

1. 差值绝对值 < 5%：低
2. 5% <= 差值绝对值 < 12%：中
3. 差值绝对值 >= 12%：高

阈值后续可配置。

## 5.4 场景修正

### 功能说明

展示模型在基础概率之外额外考虑的现实比赛场景因素，用于解释预测为什么发生偏移。

### 当前修正因子

1. 伤病情况
2. 战术习惯
3. 联赛数据
4. 队友羁绊
5. 天气情况
6. 平均年龄
7. 比赛时段
8. 海拔适应
9. 球速节奏

### 专家优先级

根据资深球迷经验，战术习惯和平均年龄/体能权重更高，其次是伤病情况。

### 展示字段

1. 修正标题
2. 修正说明
3. 利好对象
4. 影响权重
5. 风险等级
6. 对应图标

### 交互模式

1. 场景卡片按影响权重从高到低排列。
2. 正向修正用绿色显示。
3. 负向修正用黄色或红色显示。
4. 鼠标悬停时后续可展示数据依据。
5. 点击某个场景因子，后续可在赛程表中高亮所有受该因子影响的比赛。

## 5.5 盘口热度风险

### 功能说明

识别市场热度与模型概率分歧较大的选项。

### 展示字段

1. 选项
2. 市场热度
3. 模型概率
4. 差值
5. 风险等级

### 交互模式

1. 默认展示分歧最大的 Top 3。
2. 差值高的行标红。
3. 后续支持点击某行联动竞彩校准模块。

## 5.6 夺冠概率前 10

### 功能说明

展示各球队当前夺冠概率排序。

### 展示字段

1. 排名
2. 国家/球队
3. 国旗
4. 夺冠概率
5. 横向进度条

### 交互模式

1. 点击球队后，后续可筛选该球队相关赛程。
2. 鼠标悬停显示概率变化趋势。
3. 数据更新后排序自动调整。

## 5.7 模型复盘

### 功能说明

展示模型近期预测效果，让用户判断模型是否可靠。最小可用产品阶段将“整体表现”和“分玩法命中率”合并为一个模块，减少底部信息重复。

### 展示字段

1. 整体命中率
2. 盈利率或模拟收益率
3. 均值回报率
4. 近 30 天趋势图
5. 胜平负命中率
6. 受让球胜平负命中率
7. 半全场命中率
8. 比分误差命中率
9. ROI 或回报指标

### 交互模式

1. 默认展示近 30 天。
2. 后续可切换近 7 天、近 30 天、全周期。
3. 趋势图可按市场类型切换：胜平负、受让球胜平负、半全场。
4. 点击某个预测类型后，后续可进入对应复盘列表。
5. 颜色区分不同市场类型。
6. 低于阈值的指标后续可标黄或标红。
7. 模型复盘属于全局观察区，点击单场比赛时不跟随变化。

## 5.8 专家因子权重

### 功能说明

展示预测模型对不同因子的权重分布，让用户理解模型依据。

2. 平均年龄/体能：20%
3. 伤病情况：18%
4. 联赛数据：12%
5. 队友羁绊：10%
6. 天气情况：6%
7. 比赛时段：4%
8. 海拔适应：3%
9. 球速节奏：2%

### 交互模式

1. 最小可用产品阶段只展示权重。
2. 后续可支持用户拖动调整权重。
3. 调整权重后，页面重新计算单场预测。
4. 提供“恢复默认权重”按钮。

## 6. 预测模型需求

### 6.1 模型分层

预测逻辑分为四层：

1. 数据层
2. 概率计算层
3. 战术博弈层
4. 智能/专家校准层

### 6.2 数据层

需要输入：

1. 球队基础实力
2. FIFA 排名或 Elo
3. 近期战绩
4. 进球与失球
5. 球员阵容
6. 伤病停赛
7. 平均年龄
8. 体能状态
9. 队内配合时间
10. 联赛强度
11. 场地与天气
12. 比赛时间
13. 海拔
14. 竞彩赔率

数据来源后续讨论补充。

### 6.3 概率计算层

基础模型建议：

1. Elo 计算球队强弱差。
2. 泊松分布计算比分概率。
3. 将比分矩阵汇总为胜平负概率。
4. 计算 xG 预期进球。
5. 输出最可能比分 Top N。

### 6.4 战术博弈层

战术类型：

1. 控制型
2. 逼抢型
3. 反击型
4. 定位球型

战术修正示例：

1. 控制型对弱队提升控球稳定性。
2. 逼抢型受体能和年龄影响更大。
3. 反击型增加爆冷概率。
4. 定位球型提高低比分胜率。

### 6.5 智能/专家校准层

校准输入：

1. 教练风格
2. 首发阵容
3. 伤病变化
4. 球队新闻
5. 赛前发布会
6. 赔率异动
7. 专家经验权重

校准输出：

1. 最终胜平负概率
2. 最终比分预测
3. 冷门风险
4. 关键解释语句

## 8. 交互细节

### 8.1 页面加载

1. 默认加载总览页。
2. 默认选择第一场重点比赛。
3. 单场联动区使用默认选中比赛作为分析对象。
4. 全局观察区使用赛事阶段或全局模型数据作为展示对象。
5. 若数据未加载完成，显示骨架屏或 loading 状态。
6. 若数据加载失败，显示错误提示和重试按钮。

### 8.2 比赛切换

触发方式：

1. 点击赛程表行。
2. 后续可点击球队、夺冠榜或搜索结果。

联动范围：

1. 焦点战预测更新。
2. 竞彩校准更新。
3. 场景修正更新。
4. 盘口热度风险更新。
5. 选中行高亮更新。

不联动范围：

1. 夺冠概率前 10 不随单场点击变化。
2. 模型复盘不随单场点击变化。
3. 专家因子权重不随单场点击变化，除非用户进入权重编辑模式。
4. 顶部全局状态不随单场点击变化，只在刷新或筛选变化时更新。

联动数据边界：

```text
selectedMatchId
  -> 焦点战预测
  -> 当前比赛竞彩市场
  -> 当前比赛场景修正
  -> 当前比赛盘口热度风险
  -> 赛程当前选中态

globalTournamentState
  -> 夺冠概率
  -> 模型复盘
  -> 专家因子权重
  -> 数据状态与更新时间
```

### 8.3 刷新

点击刷新按钮后：

1. 请求最新赛程。
2. 请求最新竞彩数据。
3. 请求最新伤病/阵容/天气数据。
4. 更新顶部“最后更新时间”。
5. 若刷新失败，保留旧数据并提示“更新失败，已显示上次数据”。

### 8.4 筛选

筛选改变后：

1. 赛程列表刷新。
2. 若当前选中比赛不在筛选结果中，则自动选中筛选结果第一场。
3. 若没有比赛，显示空状态。

### 8.5 风险提示

高风险条件：

1. 市场热度与模型概率差值过大。
2. 临场伤病未确认。
3. 首发阵容未公布。
4. 天气、海拔或比赛时段对某队影响明显。
5. 赔率短时间剧烈变化。

展示方式：

1. 红色风险标签。
2. 风险表格置顶。
3. 后续可添加弹窗解释。

## 9. 页面视觉要求

### 9.1 整体风格

1. 深色数据控制台。
2. 专业、紧凑、高信息密度。
3. 避免营销式大 Hero。
4. 使用清晰网格布局。
5. 模块边框弱化但可识别。

### 9.2 色彩

1. 背景：深青黑。
2. 主强调：绿色，用于命中、利好、正向趋势。
3. 警示：红色，用于风险、负向偏差、赔率异常。
4. 中性：黄色/琥珀色，用于中风险和提醒。
5. 辅助：蓝色，用于客队或次要指标。

### 9.3 布局要求

1. 上方三块模块底部需要视觉对齐。
2. 场景修正与赛程模块底部需要对齐。
3. 底部四个模块高度一致。
4. 表格文本不允许溢出遮挡。
5. 小屏幕下允许模块纵向堆叠。

## 10. 非功能需求

### 10.1 性能

1. 首屏加载尽量控制在 2 秒内。
2. 数据刷新不阻塞页面操作。
3. 表格数据较多时支持分页或虚拟滚动。

### 10.2 兼容性

1. 支持 Chrome 最新版。
2. 支持 1440px 及以上桌面宽度。
3. 支持 1024px 平板横屏。
4. 移动端允许简化布局。

### 10.3 可维护性

1. 比赛数据、市场数据、因子数据分离。
2. 预测逻辑与展示组件分离。
3. 后续接入接口时不大改 UI。
4. 组件命名清晰。

### 10.4 合规说明

页面必须明确展示：

```text
数据仅供分析复盘，不构成任何投注建议。
```

产品不提供投注入口、不承诺收益、不引导下注。

## 14. 风险与注意事项

1. 竞彩数据来源可能涉及授权，需要后续确认合法合规渠道。
2. 预测结果不能表述为确定性结论。
3. 伤病、首发、天气等临场数据波动较大，需要显示更新时间。
4. 模型命中率需要基于真实历史数据计算，不能用模拟数据对外宣传。
5. 移动端信息密度过高，后续可能需要单独设计移动版。
